# 2. 자동차 ECU 환경의 소프트웨어 업데이트를 위한 해시 함수 성능 평가 연구

## 개요

본 연구는 자동차 ECU(Environment Control Unit) 환경에서의 소프트웨어 업데이트 시, 무결성 검증을 위한 해시 함수의 성능을 비교·평가하기 위해 수행되었습니다.

특히 **BLAKE3** 알고리즘이 리소스가 제한된 임베디드 환경에서 **SHA-256 대비 처리 속도, CPU 점유율, 메모리 사용량 측면에서 얼마나 효율적인지**를 검증하였습니다.

---

## 연구 배경

- 자동차 ECU 소프트웨어 업데이트의 중요성이 증가하면서, 업데이트 파일의 **무결성 검증**이 필수화됨.
- 현재 대부분의 환경에서 **SHA-256**이 표준 해시 알고리즘으로 사용되지만,
임베디드 환경에서는 처리 속도와 메모리 사용량 측면에서 부담이 큼.
- **BLAKE3**는 2008년 발표된 BLAKE 시리즈의 최신 버전으로, **SIMD 병렬 처리** 및 **Merkle Tree 구조**를 통해 빠른 연산과 효율적 자원 사용을 지원.

---

## 연구 가설

- BLAKE3는 SIMD 기반 병렬 처리 구조로 인해 ARM CPU 기반 임베디드 환경에서
SHA-256, SHA3-256, BLAKE2S, BLAKE2B보다 **빠르고 효율적인 성능**을 보일 것이다.

---

## 알고리즘 구조 비교

### BLAKE3의 핵심 구조

1. **Parallel Processing (병렬 처리)**
입력 데이터를 여러 청크(chunk)로 나누어 동시에 처리 → 처리 속도 향상.
2. **Merkle Tree 구조**
각 청크의 해시 결과를 병합하여 최종 해시 코드 생성 → 연산 효율 개선.
3. **SIMD 최적화 (NEON 기반)**
ARM CPU의 벡터 연산 유닛(SIMD)을 적극 활용하여 하나의 명령어로 다중 연산 수행.

---

## 실험 방법

- **파일 크기:** 1MB, 10MB, 100MB, 500MB, 1000MB
- **대상 알고리즘:** SHA-256, SHA3-256, BLAKE2S, BLAKE2B, BLAKE3
- **환경:** Raspberry Pi 3B+ (ARM CPU, Ubuntu 22.04 LTS 64-bit)
- **측정 항목:**
    - 해싱 시간 (초)
    - CPU 점유율 (getrusage)
    - 메모리 사용량 (VmRSS, Peak / Average)
- **측정 방법:**
각 알고리즘으로 암호화된 파일을 50회 반복 해싱하고 평균값 도출.

---

## 실험 결과

### 1. 해싱 시간

- BLAKE3는 파일 크기와 무관하게 일정하고 빠른 처리 속도를 유지.
- 1000MB 기준 **SHA-256 대비 약 60%의 처리 시간 단축**.
- SIMD 연산을 활성화할 경우 **추가로 35% 성능 향상**을 보임.

### 2. 메모리 사용량

- BLAKE3는 SHA-256 대비 **평균 68% 메모리 사용량 감소**.
- BLAKE2S, BLAKE2B보다도 효율적인 메모리 관리 특성 확인.

### 3. CPU 점유율

- SHA-256 대비 **약 13% 낮은 CPU 점유율** 기록.
- 병렬 연산과 경량 구조 덕분에 연산 효율성이 높음.

---

## 결론

- **BLAKE3**는 모든 파일 크기에서 가장 빠른 해싱 속도를 기록하며,
메모리와 CPU 자원이 제한된 **자동차 ECU 및 임베디드 환경에 적합**한 해시 알고리즘임을 입증.
- **평균 1.33MB 낮은 메모리 사용량**, **60% 이상 처리 시간 단축**, **낮은 CPU 점유율**로
실시간 검증 및 OTA(Over-The-Air) 업데이트 시스템에서 활용 가능성이 높음.

---

## 향후 연구 방향

- 실제 차량 ECU 환경에서의 **전력 소비량 측정** 및
멀티스레드 기반 병렬 처리 효율성 검증.
- **멀티 스레드 환경에서의 메모리 증가량 분석** 및**BLAKE3 컨텍스트 최적화 방안** 연구.
- 향후 **CP-ABE 기반 암호화 시스템**이나 **블록체인 무결성 검증 구조**와의 연계 연구 계획.
