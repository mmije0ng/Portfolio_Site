## *Overview*

---

- **목적**: 파편화된 지역별 경제 뉴스를 수집·분석하여 지역별 경제 온도를 시각화하고, 시장 지표와의 상관관계를 규명함.
- **핵심 가치**: 실시간 데이터 파이프라인 구축 + NLP 기반 감성 분석 + 인터랙티브 시각화 대시보드.
- **주요 특징**: 12개 권역 언론사 자동 스크래핑, KoBERT 기반 경제 체감 지수 산출, KOSPI/KOSDAQ 상관계수($r$) 분석.

## ***Contributions***

---

**1. 고성능 뉴스 스크래핑 파이프라인 구축**

- **멀티 스레드 도입**: `ThreadPoolExecutor`를 활용한 병렬 처리를 통해 수집 속도를 **단일 스레드 대비 약 10배 개선**.
- **안정성 확보**: 지수 백오프(Exponential Backoff) 로직을 적용하여 서버 부하 및 차단 문제를 방지하고 최대 3회 재시도 로직 구현.
- **데이터 표준화**: 언론사별 상이한 HTML 구조에 대응하는 셀렉터 리스트 설계 및 상대 시간을 절대 날짜 형식으로 변환.

**2. 정교한 데이터 전처리 및 정제**

- **노이즈 제거**: BeautifulSoup의 `decompose()` 및 정규표현식을 활용해 광고, 기자 메일, 저작권 문구 등 분석 방해 요소 완벽 제거.
- **인코딩 최적화**: `utf-8-sig` 및 `chardet`을 활용해 언론사별 인코딩 깨짐 현상을 해결하고 데이터 정합성 확보.

**3. 인터랙티브 웹 대시보드 구현**

- **데이터 시각화**: 지역별 뉴스 밀도와 감성 지수를 결합한 시각화 UI 개발.
- **실시간 피드**: 분석 결과의 근거가 되는 원본 기사를 즉시 확인할 수 있는 맞춤형 뉴스 피드 기능 구현.

## *Problem Solving*

---

![미니프로젝트1 4조 Senti-Logic 프로젝트 발표 (1).png](%EB%AF%B8%EB%8B%88%ED%94%84%EB%A1%9C%EC%A0%9D%ED%8A%B81_4%EC%A1%B0_Senti-Logic_%ED%94%84%EB%A1%9C%EC%A0%9D%ED%8A%B8_%EB%B0%9C%ED%91%9C_(1).png)

### 1. 웹 스크래핑 효율성 및 안정성 최적화

- **문제**: 11개 지역 언론사의 방대한 뉴스 데이터를 단일 스레드로 수집 시 과도한 시간 소요 및 서버 측의 연결 차단(Connection Refused) 발생.
- **해결**:
    - **병렬 처리 도입**: `ThreadPoolExecutor`를 활용한 멀티 스레딩 환경을 구축하여 단일 스레드 대비 **수집 속도를 약 10배 향상**.
    - **지수 백오프(Exponential Backoff)**: 네트워크 오류나 서버 부하 발생 시 대기 시간을 점진적으로 늘리며 최대 3회까지 재시도하는 로직을 적용해 수집 중단 현상 방지.
    - **연결 최적화**: `requests`와 `urllib` 사용 시 `keep-alive`를 통한 연결 재사용으로 통신 오버헤드를 최소화하고 SSL 인증서 검증 무시 설정을 통해 안정성 확보.

### 2. 대규모 데이터 저장 및 아키텍처 개선

- **문제**: 수집 데이터 양이 증가함에 따라 단일 파일 저장 시의 성능 저하 우려.
- **해결**:
    - **2단계 저장 프로세스**: 수집 즉시 원본 데이터를 포함한 세부 내용을 CSV(1차)로 백업한 후, DB 스키마에 맞춰 정제된 주요 데이터만 SQLite3(2차)에 마이그레이션하는 안정적인 적재 구조 구축.

![미니프로젝트1 4조 Senti-Logic 프로젝트 발표 (2).png](%EB%AF%B8%EB%8B%88%ED%94%84%EB%A1%9C%EC%A0%9D%ED%8A%B81_4%EC%A1%B0_Senti-Logic_%ED%94%84%EB%A1%9C%EC%A0%9D%ED%8A%B8_%EB%B0%9C%ED%91%9C_(2).png)

### 3. 한글 깨짐(Encoding) 오류 및 데이터 정합성 해결

- **문제**: `requests` 라이브러리가 헤더 정보 부족으로 인코딩을 서유럽 언어(ISO-8859-1) 등으로 오판하여 기사 본문이 깨지는 현상 발생.
- **해결**:
    - **인코딩 강제 교정**: 응답 단계에서 인코딩 방식을 `utf-8`로 선제 지정하는 공통 함수 적용.
    - **2단계 폴백(Fallback) 로직**: 언론사별 고유 인코딩을 우선 시도하되, 실패하거나 선언이 누락된 경우 `utf-8` 및 `utf-8-sig`로 강제 전환하는 예외 처리 프로세스 구축.
    - **텍스트 정규화**: `chardet` 라이브러리를 병행 사용하여 인코딩 감지 정확도를 높임.

![미니프로젝트1 4조 Senti-Logic 프로젝트 발표.png](%EB%AF%B8%EB%8B%88%ED%94%84%EB%A1%9C%EC%A0%9D%ED%8A%B81_4%EC%A1%B0_Senti-Logic_%ED%94%84%EB%A1%9C%EC%A0%9D%ED%8A%B8_%EB%B0%9C%ED%91%9C.png)

### 4. 비정형 데이터 정제 및 파싱 구조 설계

- **문제**: 언론사마다 상이한 HTML 구조와 기사 본문에 포함된 광고, 기자 메일, 저작권 문구 등 노이즈 데이터 포함.
- **해결**:
    - **유연한 셀렉터 설계**: 사이트별로 다른 구조에 대응하기 위해 `selectors`를 리스트 형태로 관리하고, 후보 태그 중 존재하는 데이터를 유동적으로 가져오는 구조 설계.
    - **노이즈 제거 파이프라인**: BeautifulSoup의 `decompose()` 함수로 불필요한 태그를 물리적으로 제거하고, 정규표현식(`clean_text`)을 통해 텍스트 노이즈를 2차 정제.
    - **날짜 표준화**: 'n시간 전', '어제' 등 상대적인 시간 표기를 분석 가능한 절대적인 날짜 데이터 형식으로 변환하여 DB 정합성 확보.
