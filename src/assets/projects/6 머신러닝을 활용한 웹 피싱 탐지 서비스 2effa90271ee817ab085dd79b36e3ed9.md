## ***Overview***

---

**머신러닝 모델을 기반으로 웹 사이트 URL을 분석하여 피싱 여부를 판단하는 서비스**

- XGBoost 기반의 머신러닝 모델을 활용해 URL을 분석하고 피싱 여부를 판별
- 사용자의 접근성을 높이기 위해 **Chrome 확장 프로그램**과 **웹 애플리케이션** 형태의 인터페이스 제공

**확장 프로그램**

- URL을 실시간으로 분석하여 피싱 사이트로 의심될 경우 경고 메시지를 표시.
- 사용자가 판별하고자 하는 웹 사이트의 **URL에 마우스 오버**할 때, **해당 사이트가 피싱 사이트인지 아닌지를 즉시 판별**
- 사용자는 매번 별도의 피싱 판별 웹 사이트를 방문하지 않고도 확장 프로그램을 통해 실시간으로 피싱 판별 결과를 받을 수 있어 편의성 향상

**웹 사이트**

- 확장 프로그램에서 자세히 보기를 클릭하면, 웹 사이트로 이동하여 피싱 판별 대상 웹 사이트의 **상세 정보(IP 주소, 국가, 지역, ISP 이름, VPN 사용 여부)**와 **피싱 판별 이유를 표시**
- 웹 사이트에 직접 접속하여 **URL을 입력**하여도 피싱 여부를 즉시 확인할 수 있음
- 머신러닝 모델을 통해 판별된 **피싱 확률**과 **피싱 판별에 영향을 미친 주요 피처**들을 확인할 수 있어 사용자의 이해도와 신뢰성을 높임.

**CI/CD 및 배포 인프라 구축**

프론트엔드&백엔드 CI/CD 자동화 환경을 구축하여 **모델 학습–빌드–배포 과정을 완전 자동화**함으로써 안정적이고 지속적인 배포가 가능한 구조를 설계하였습니다.

1. **GitHub Actions**
    - main 브랜치에 코드가 **push**되면 자동으로 **빌드 및 테스트 워크플로우**가 실행되도록 구성
    - 프론트엔드는 **React 빌드 파일**을 S3 버킷에 자동 업로드하고,
        
        백엔드는 **AWS CodeDeploy**를 통해 EC2 인스턴스로 자동 배포
        
2. **AWS CodeDeploy + S3 + EC2 파이프라인**
    - GitHub Actions에서 생성된 빌드 산출물을 **S3 Bucket**에 업로드
    - CodeDeploy가 S3에서 빌드 파일을 받아 **EC2 인스턴스**에 배포
    - Flask 서버와 MySQL(Amazon RDS)을 포함한 애플리케이션 스택을 자동 재시작 및 헬스체크
3. **AWS CloudFront + Route 53 + ACM (보안/도메인 관리)**
    - S3 정적 리소스를 **CloudFront**를 통해 전 세계 CDN 캐싱으로 가속화
    - **Route 53**을 이용해 도메인과 IP 매핑
    - **AWS Certificate Manager(ACM)**을 통해 SSL 인증서를 자동 관리하여
        
        HTTPS 통신 기반의 안전한 웹 서비스 제공
        
4. **Application Load Balancer (ALB)**
    - EC2와 RDS 인스턴스 사이의 트래픽을 **로드 밸런싱**하여 안정적인 서비스 처리
    - 다수의 인스턴스가 운영될 경우에도 장애 발생 시 자동으로 트래픽 분산
5. **Amazon RDS (MySQL)**
    - 모델 학습 결과, 탐지 로그, 사용자 요청 데이터를 저장
    - Flask API에서 SQLAlchemy ORM을 통해 접근하며, 트랜잭션 기반 데이터 관리로 무결성 및 일관성 보장

## *Problem Solving*

### **1. 데이터셋 구성 및 피처 엔지니어링**

![image.png](image%2022.png)

![image.png](image%2023.png)

정상 URL과 피싱 URL을 약 1:1 비율로 수집하여 총 **30개의 피처(URL, Content, Domain 기반)** 로 구성된 데이터셋을 구축했습니다.

각 피처는 `having_IP_Address`, `URL_Length`, `Prefix_Suffix`, `HTTPS_token`, `Request_URL` 등 URL 구조적 특징뿐만 아니라,

도메인 등록 기간(`Domain_registration_length`), 웹 트래픽(`Web_traffic`) 등 신뢰도 관련 정보까지 포함하도록 설계했습니다.

모델의 효율성과 일반화 성능을 높이기 위해 **피처 중요도 분석(Feature Importance Analysis)** 과 **상관관계 분석(Correlation Analysis)** 을 수행했습니다.

XGBoost 모델을 활용하여 각 피처의 중요도를 계산하고, Pandas·Scikit-learn·Matplotlib을 이용해 중요도 시각화를 진행했습니다.

그 결과, 중요도가 낮거나 유사한 상관관계를 보이는 **6개의 피처(Web_traffic, Page_Rank, Links_pointing_to_Page, Statistical_report, Domain_registration_length, Abnormal_URL)** 를 제거했습니다.

이후 교차 검증(Cross Validation)을 통해 피처 제거 전후의 모델 성능을 비교한 결과, **평균 정확도 0.9548**로 기존 대비 성능 저하 없이 모델 단순화에 성공했습니다.

이 과정을 통해

1. **과적합 방지(Overfitting Prevention)** — 불필요한 피처를 제거하여 새로운 데이터에 대한 일반화 성능 향상,
2. **모델 단순화(Model Simplification)** — 학습 및 예측에 필요한 연산량 감소로 학습 시간 단축과 계산 효율성 향상,
3. **모델 안정성 확보(Model Stability)** — 피처 간 중복 정보 제거를 통해 예측의 일관성과 신뢰성 향상 등의 효과를 얻었습니다.

최종적으로, 중요도 기반 피처 셀렉션을 통해 모델의 **성능(Accuracy 95.48%)** 과 **해석 가능성(Explainability)** 을 모두 확보한 최적의 피처 집합을 구성했습니다.

### **2. 피처 분류**

![image.png](image%2024.png)

피싱 사이트 탐지를 위해 웹사이트의 특성을 **URL, Domain, Content 기반의 3가지 범주로 피처를 분류**했습니다.

이러한 피처들은 웹 주소의 구조적 특징, 도메인의 신뢰도, 그리고 HTML 콘텐츠 내 행위적 요소를 종합적으로 분석하여 **피싱 여부를 높은 정밀도로 판별**할 수 있도록 설계되었습니다.

- **URL 기반 피처**는 URL 길이(`url_length`), 비표준 포트 사용(`port`), `@`, `//`,  등의 특수문자 포함 여부 등을 포함합니다.
    
    이는 피싱 사이트가 일반적으로 긴 URL을 사용하거나 특수문자를 혼용해 사용자를 속이는 경향이 있다는 점에 착안한 것입니다.
    
- **Domain 기반 피처**는 도메인의 수명(`age_of_domain`), SSL 인증서의 유효성(`ssl_final_state`), 서브도메인 포함 여부(`having_sub_domain`) 등을 분석합니다.
    
    이를 통해 피싱 사이트가 짧은 수명의 도메인이나 유효하지 않은 인증서를 사용하는 특성을 포착할 수 있습니다.
    
- **Content 기반 피처**는 웹 페이지의 소스 코드 내에서 `iframe`, `popup_window`, `form` 태그, 이메일 리다이렉션(`submitting_to_email`), 마우스 오버 이벤트(`on_mouseover`) 등의 존재 여부를 확인합니다.
    
    이러한 피처는 사용자의 입력 정보를 탈취하거나 악성 행위를 수행하는 스크립트 패턴을 탐지하는 데 활용됩니다.
    

피처 설계는 기존 연구([Mohammad et al., *Phishing Websites Features*, University of Huddersfield, 2015](https://eprints.hud.ac.uk/id/eprint/24330/6/MohammadPhishing14July2015.pdf))를 기반으로 수행하였으며, 이를 확장하여 실제 피싱 사이트 데이터셋에 적용함으로써 **구조적(Structural)** + **행동적(Behavioral)** 분석이 결합된 정교한 탐지 피처셋을 구축했습니다.

### **3. 머신러닝 모델 선정 및 성능 평가**

![image.png](image%2025.png)

![image.png](image%2026.png)

![image.png](image%2027.png)

피싱 URL 탐지를 위해 **XGBoost, MLP, Random Forest, SVM** 총 4가지 머신러닝 모델을 비교·분석했습니다.

각 모델은 동일한 전처리된 데이터셋을 기반으로 학습되었으며, **정확도(Accuracy)**, **정밀도(Precision)**, **재현율(Recall)**, **F1-score**, **교차 검증 평균 정확도**를 지표로 평가했습니다.

그 결과,

- **MLP(Multi-Layer Perceptron)** 모델은 가장 높은 정확도(0.9765)와 F1-score(0.9760)를 기록하며 복잡한 패턴 인식에 강점을 보였습니다.
- 반면 **XGBoost**는 정확도 0.9710, F1-score 0.9704로 약간 낮지만, **학습 속도**, **모델 해석 가능성(Feature Importance)**, **과적합 방지 성능**에서 가장 우수한 결과를 보였습니다.

**XGBoost는 다음과 같은 이유로 최종 모델로 선정되었습니다.**

1. **블랙박스 모델 vs 트리 기반 구조 모델 비교 분석**
    - MLP는 내부 연산이 복잡한 블랙박스 모델이라 판별 근거를 알기 어려웠습니다.
    - 반면 XGBoost는 트리 기반 구조로 '피처 중요도'를 통해 어떤 데이터 속성이 판별에 결정적 영향을 주었는지 명확히 설명할 수 있었습니다.
2. **정확도와 일반화 성능의 균형**
    - XGBoost는 Gradient Boosting 기반의 앙상블 모델로, 여러 약한 결정트리(Weak Learner)를 결합하여 오차를 줄이는 구조입니다.
    - 교차 검증 결과 평균 정확도 0.9713을 기록하며, 다른 모델 대비 편차가 가장 적었습니다.
    - 이는 데이터에 과적합되지 않고 **안정적인 일반화 능력**을 확보했음을 의미합니다.
3. **하이퍼파라미터 튜닝 최적화 및 성능 향상**
    - `GridSearchCV`를 통해 `n_estimators`, `learning_rate`, `max_depth`, `min_child_weight`, `subsample`, `colsample_bytree` 등 주요 파라미터를 최적화했습니다.
    - 최적 조합(`n_estimators=300`, `learning_rate=0.2`, `max_depth=5`, `subsample=0.8`)을 적용한 결과, **테스트 정확도가 0.9710 → 0.9713으로 향상**되었습니다.
    - 이는 모델의 복잡도를 높이지 않으면서도 학습 효율을 개선한 결과로, 성능과 계산 자원의 균형을 동시에 달성했습니다.
4. **피처 셀렉션 기반 효율성 향상**
    - 피처 중요도 분석을 통해 중요도가 낮은 6개 피처를 제거한 뒤 학습 속도를 비교한 결과,
        
        **학습 시간은 2.882초 → 1.082초로 약 62% 단축**, 정확도는 0.9710에서 0.9538로만 감소했습니다.
        
    - 즉, 약간의 정확도 손실로 **모델 경량화와 계산 효율성 최적화**를 동시에 달성했습니다.
5. **혼동 행렬 기반 예측 신뢰성 검증**
    - 혼동 행렬 분석 결과, 정상 URL(3778건)과 피싱 URL(4807건)을 명확히 구분했으며,
        
        오탐(False Negative) 비율은 2.3% 이하로 유지되었습니다.
        
    - 이는 모델이 피싱 URL을 놓치지 않으면서도 불필요한 경보(False Alarm)를 최소화했음을 보여줍니다.
6. **모델 해석력(Explainability)**
    - XGBoost의 Feature Importance 시각화를 통해, `SSL_final_state`, `URL_of_Anchor`, `web_traffic` 등이 탐지 정확도에 가장 큰 영향을 미친 것으로 분석되었습니다.
    - 이를 기반으로 피싱 탐지의 핵심 요인을 식별할 수 있었으며, **모델의 투명성과 설명 가능성(Explainability)** 이 확보되었습니다.

결론적으로, **XGBoost는 정확도 97.1%, F1-score 0.97, 학습 시간 1.08초**로 정확성·효율성·해석 가능성 측면에서 가장 균형 잡힌 모델로 평가되었습니다.

이 모델은 피처 셀렉션 및 하이퍼파라미터 최적화를 통해 **실시간 탐지 환경에서도 안정적이고 빠른 피싱 탐지 성능을 제공할 수 있는 최적의 솔루션**으로 확립되었습니다.
